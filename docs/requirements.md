# CoffeeVision 要件定義

## アプリ概要

| 項目 | 内容 |
|------|------|
| アプリ名 | CoffeeVision |
| プラットフォーム | iOS（先行リリース）/ Android（後追い） |
| ターゲット OS | iOS 26 以上 / Android 16（API 36）以上 |
| ターゲットユーザー | カフェ巡りを趣味とし、訪問記録 / 飲んだコーヒーの情報を残したいユーザー |
| コンセプト | カフェでの体験（カフェ・コーヒー・フード）を 1 つの「Visit（訪問）」として記録し、後で見返したり振り返ったりできるアプリ |

### 1. 認証・アカウント

クラウド同期のため Firebase Authentication を採用する。匿名認証から開始し、後でメール / SNS にアップグレード可能とする。

| # | 機能 | 優先度 | 備考 |
|---|------|--------|------|
| 1-1 | 匿名サインイン | ◎ | 初回起動時に自動で匿名アカウントを作成 |
| 1-2 | メール / SNS（Apple / Google）でのアップグレード | ○ | 匿名アカウントの履歴を引き継ぐ |
| 1-3 | サインアウト | ○ | |
| 1-4 | アカウント削除 | ○ | Firestore のドキュメントも削除する |

### 2. Visit（訪問記録）管理

Visit とはカフェへの 1 回の訪問記録の単位。1 Visit には 0..N の CoffeeItem と 0..N の FoodItem が紐づく。

**Visit フィールド**

| フィールド | 型 | MVP | 備考 |
|----------|---|-----|------|
| カフェ情報 | `Cafe`（Places API 由来） | ◎ | place_id・名称・住所・座標・写真参照などをスナップショット保存 |
| 訪問日 | `LocalDate` | ◎ | デフォルトは今日 |
| 内装の雰囲気 | `String`（自由入力） + タグ候補 | ◎ | 「落ち着き」「広い」「写真映え」等のタグも将来検討 |
| 総合評価 | `Int`（1..5）または `Double`（0.5 刻み） | ◎ | UI は星 5 段階 |
| メモ | `String`（複数行） | ◎ | |
| 写真 | 複数の `Photo`（最大 N 枚） | ◎ | 端末ローカル（Documents 配下）に保存。iCloud Backup でバックアップ |
| CoffeeItem 一覧 | `List<CoffeeItem>` | ◎ | 1..N が一般的だが 0 件もあり得る |
| FoodItem 一覧 | `List<FoodItem>` | ◎ | 0 件もあり得る |

**Visit 機能**

| # | 機能 | 優先度 | 備考 |
|---|------|--------|------|
| 2-1 | Visit の新規作成（カフェ検索 → 詳細入力） | ◎ | Places API でカフェを検索 |
| 2-2 | Visit の編集 | ◎ | |
| 2-3 | Visit の削除 | ◎ | |
| 2-4 | Visit 一覧の表示（時系列） | ◎ | 最新順 |
| 2-5 | Visit 詳細の表示 | ◎ | |
| 2-6 | お気に入り Visit のピン留め | ○ | |
| 2-7 | カフェ別の Visit 集約表示 | ○ | 同一 place_id をまとめる |

### 3. CoffeeItem 管理

Visit に紐づく 1 杯のコーヒー記録。

**CoffeeItem フィールド**

| フィールド | 型 | MVP | 備考 |
|----------|---|-----|------|
| メニュー名 | `String` | ◎ | 「本日のコーヒー」「エチオピア イルガチェフェ」など |
| 抽出方法 | `enum BrewMethod` | ◎ | エスプレッソ / ハンドドリップ / ネルドリップ / フレンチプレス / エアロプレス / サイフォン / コールドブリュー / その他 |
| 産地 | `String?`（国名 / エリア） | ◎ | |
| 品種 | `String?`（自由入力 + 推奨候補） | ○ | ティピカ / ブルボン / ゲイシャ / SL28 等 |
| 精製方法 | `enum ProcessingMethod?` | ○ | ナチュラル / ウォッシュト / ハニー / アナエロビック / その他 |
| 焙煎度 | `enum RoastLevel?` | ◎ | ライト / シナモン / ミディアム / ハイ / シティ / フルシティ / フレンチ / イタリアン |
| カップ | `String?`（使用されたカップの種類 / ブランド等のメモ） | ○ | |
| 評価 | `Int`（1..5） | ◎ | |
| メモ | `String?` | ○ | 風味・所感の自由記述 |

**機能**

| # | 機能 | 優先度 |

| 3-1 | Visit 内での CoffeeItem 追加 / 編集 / 削除 | ◎ |
| 3-2 | CoffeeItem 単独詳細表示 | ○ |
| 3-3 | 抽出方法 / 産地 / 焙煎度などによるフィルタ | △ |

## 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 一覧表示は 60fps を維持する。画像はサムネイルキャッシュで遅延読み込み |
| アクセシビリティ | VoiceOver 対応・Dynamic Type 対応・最小タップ領域 44pt |
| オフライン | 完全オフライン動作可能（Firestore のオフライン永続化、ローカル SQLDelight） |
| 同期 | オンライン復帰時に Firestore が自動で同期する。独自の同期キューは持たない |
| データ保存 | ローカル: SQLDelight + 写真ファイル（Documents 配下） / クラウド: Firestore（写真本体は対象外） |
| セキュリティ | Firestore Security Rules で `uid` に基づくアクセス制御。API キーは難読化 |
| プライバシー | 位置情報の利用は Places 検索時のみ。常時取得しない |
| API キー管理 | `local.properties` から `BuildConfig` 経由で注入。リポジトリにコミットしない |

## データモデル概要

| エンティティ | 説明 | 関連 |
|-------------|------|------|
| `Visit` | 1 回のカフェ訪問 | `Cafe`（埋め込み）/ `Photo`（多）/ `CoffeeItem`（多）/ `FoodItem`（多） |
| `Cafe` | Places API 由来のカフェ情報スナップショット | `place_id` をキーに集約可能 |
| `CoffeeItem` | 1 杯のコーヒー記録 | `Visit` に従属 |
| `FoodItem` | フード記録 | `Visit` に従属 |
| `Photo` | 写真への参照 | Documents 配下からの相対ファイル名（クラウド同期対象外） |

詳細なフィールド定義・SQLDelight / Firestore スキーマは [`data-model.md`](./data-model.md) を参照。
